## 分块策略

### 分块参数
```python
{
    "chunk_size": 500,      # 每块约500字符
    "chunk_overlap": 100,   # 块间重叠100字符
    "separators": ["\n\n", "\n", "。", "！", "？", "，", " ", ""]
}
```

### 分块原则
1. **保持章节结构**：先按章节分割，再在每个章节内分块
2. **语义完整性**：优先在句号、段落处切分，不在词语中间切分
3. **上下文保留**：块间重叠100字符，保证检索时上下文连贯
4. **过滤短块**：少于50字符的块丢弃（避免标题、页码等碎片）

### 元数据设计
每块包含以下元数据：
```json
{
  "source": "美国儿科学会育儿百科",
  "author": "塔尼娅·奥尔特曼/美国儿科学会",
  "chapter": "第一章 新生儿护理",
  "chunk_id": 5,
  "book": "book0.txt"
}
```

## 优化建议

### 1. 提高检索效果
- 添加同义词扩展（如"发烧"="发热"）
- 对问答对进行人工标注，微调embedding模型

### 2. 处理长文档
- 对于超过500字的段落，优先在子标题处分割
- 添加摘要索引，先检索摘要再定位详细内容

### 3. 质量控制
- 定期检查检索结果的相关性
- 根据用户反馈调整分块大小
